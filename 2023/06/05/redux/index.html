redux(react-redux)

发表于2023-06-05|更新于2023-06-06|react
|阅读量:

简介

js应用的状态容器，提供状态管理。

使用时机

  1. 存在大量状态
  2. 多人协同

管理流程

创建容器Store, 存储状态及事件池。 组件渲染就使用store的状态， 需要修改则需要去使用 redux的事件流程来修改。

安装

安装redux: npm i -D-S redux

[旧版]创建、起步仓库

新建src/store， 新建index.js。

通过redux引入createStore来进行创建仓库。 传入reducer管理员。

reducer则是一个函数，接受state和actions两个状态、事件池两个参数。 第一次赋值给state来定义initialValue。

第一次派发在redux内部派发dispatch，给state赋予初始值。

通过调用派发dispatch来修改state，由于需要修改state，而state不可以进行直接赋值， 所以需要克隆一份来进行操作后return替换state。

[旧版]批量使用

通过context上下文全局注入。后通过获取store来全局操作

[旧版]store.getState()

通过getState()来获取store的状态。

[旧版]函数组件订阅更新-subscribe

设置一个组件更新函数，传入到subscribe订阅中，return一个卸载subscribe函数，调用可以取消订阅

优化性能不设置任何依赖项，需要useEffect来return一个释放函数，来操作优化

也可以使用useState、setState时间戳、random等 来进行订阅组件不断更新。

[旧版]class组件订阅更新-subscribe

在componentDidMount中设置subscribe订阅，可以传入this.forceUpdate

[旧版]事件派发-dispatch

通过dispatch派发事件池 来修改state从而订阅者更新组件

工程化

combineReducers-(工程模块化)

创建多个reducer， 最后合并reducer传入到store中。

派发行为标识(工程模块化)

新建actions-types.js文件，里面存在统一的行为标识常量。

actions管理

创建actionsCreator

对react-redux有意义。

对dispatch行为进行统一管理。 新建actions文件夹下设置对应module的action管理。

react-redux

解决redux在react应用中的痛点

  1. 基于上下文注入等 => 自己创建了上下文对象
  2. 事件池无需手动注入-react-redux自动处理组件更新
  3. 把需要的信息传递组件

注册提供store-Provider

获取状态connect-mapStateToProps

connect(mapStateToProps, mapDispatchToProps)(Component) 来使用状态

第一个参数参数为state，通过return 一个对象来选择 注入的状态。

使用的话则是:

利用封装好的actions集中管理来进行替换mapDispatchToProps写法， 内部调用 redux.actionsCreators(actions.xxxx, dispatch)方法来进行合并。

中间件

安装后传入applyMiddleware中

redux-logger-日志

npm i -D-S redux-logger

每次派发，输出派发日志。 对之前的state、行为、之后的state等进行调试。

redux-thunk 异步

npm i -D-S redux-thunk redux-promise

actions异步操作后修改state，直接写入async/await会报错。

return 函数，异步操作后，手动dispatch派发。

redux-promise

npm i -D-S redux-promise

不需要修改解构偶，直接使用async/await语法糖就可以实现。

redux-toolkits

redux Toolkie官方强烈推荐的Redux开发工具集。 简化redux开发。

包括配置store、定义reducer、组件更新事件池、slice状态，无需编写任何action Creator或者action types、自带redux-thunk。

搭建、起步仓库

configureStore - store建立

features - createSlice 切片

新建/store/features/xxx.js， 使用CreateSlice进行创建切片module。

name为切片名称、initialState为初始状态、reducer为切片管理员， 后export default出去xxx.reducer

获取actionCreator

通过 slicexx.actions 的解构， 解构出与 slicexx.reducer下的函数名称一致的函数。

通过thunk进行异步派发，最后手动dispatch 解构出的同步action

使用数据及派发

useSelector(state => state.slicexxx) 来获取公共状态。

useDispatch()来获取dispatch方法。

import {} from ‘features/slicexxx’中 export 的actions来进行 派发行为。
