JSX

JSX来构建视图。

也可以创建.jsx文件来代替.js文件。

表达式

  1. 变量
  2. 运算
  3. 值
  4. 循环方法

用法

行内样式变为对象。 class 必须使用 className

组件化

首字母大写

函数组件

  • 返回JSX视图(jsx元素、virtualDOM)
  • import引入(或者直接使用)标签使用。
  • props第一个参数传入函数组件。
  • 基于babel-preset-react-app转化组件为createElement格式 => 执行创建出virtualDOM => render函数转化为真实DOM
  • 插槽也是作为children 传入给组件

缺陷：
静态组件。 组件数据不会修改视图。 除非重新调用函数组件。

class组件

  • 基于Class继承React.Component来创建动态组件
  • 等同于 React.Component.call(this—Parent组件实例), 添加私有属性： props/context/refs/updater
  • constructor必须调用super()

渲染逻辑

  1. render函数 每次调用组件都会创建一个新的实例，所以不会相互影响。
  2. 初始化属性 && 规则校验。
  3. 初始化状态(触发视图更新), 默认为null
  4. render()
  5. 触发钩子函数

数据驱动视图

  1. 通过state的 setState({updateVariable}) 可部分修改
  2. this.forceUpdate() 强制更新视图

组件更新逻辑

这是子组件自己修改状态时候：

  1. 触发 shouldComponentUpdate(nextProps, nextState):boolean。 返回一个Boolean，来决定是否进行视图更新。
  2. 触发UNSAFE_componentWillUpdate(nextProps, nextState)。 状态未修改
  3. 修改状态、属性
  4. render() 组件更新。
  5. 触发componentDidUpdate(prevProps)
  6. this.forceUpdate()会忽略shouldComponentUpdate()。
  7. 更新为 子组件全部更新后，更新父组件。

父组件更新导致的更新：

  1. UNSAFE_componentWillReceiveProps, 接受最新props之前。
  2. 重复上面的步骤。

hooks组件

基于函数组件和hooks函数， 让函数有状态、钩子函数成为动态化组件。

React16.8开启的新增特性。

pshouldUpdate => pwillUpdate => prender => cWillReciveProps => cShouldUpdate => cwillUpdate => crender => cdidUpdate => pdidUpdate

组件卸载

pwillUnmount => cwillUnmount => cdidUnmount => pdidUnmount

pureComponent 和 Component

pureComponent预定义了shouldComponentUpdate函数(不可以重写了)。 在钩子内， 会被新老属性、状态进行浅比较。

  • 如果有变化才更新。没变化则不需要更新。

  • Component也是可以通过shouldComponentUpdate中，浅比较来决定是否更新！也是可以实现pureComponent

ref

受控组件: 数据驱动视图(数据=>表单视图等)

非受控组件： ref获取DOM操作。

  1. 给DOM设置 ref = ‘xxx’. 通过this.refs.xxx来访问dom元素。 不推荐
  2. ref= { x => this.box2 = x}, 通过this.box2来访问dom元素。

setState进阶

局部更改和callback

this.setState([partialState], [callback])

第一个参数修改局部属性。 当此属性变化，触发**相应回调(发生在更新之后 componentDidUpdate之后)**和componentDidUpdate这个生命周期。

当shouldComponentUpdate：false, 回调依然触发，钩子不触发。 callback雷同与vue的$nextTick

更新队列(v:18.x)

调用一次setState会触发render， 那么同一个回调调用三次setState会触发三次render嘛？

不会！因为是”异步操作”，所有同步执行完后来统一执行setState

v16.x区别

v18.x无论什么时候setState都是异步的， 而v16.x 出现在定时器、事件监听器时是同步操作.

flushSync

“刷新updater更新队列”，可以取到最新的修改的状态，让队列”批处理”一次。

prevState(函数setState)

只可以通过 x + 1 来达到增加20 的需求，且更新一次

通过prevState来达到 异步操作的setState保存到同步循环次数的1。

把传入setState的回调callback放入更新队列，批处理时 来拿到 上一次return的state。

合成事件(Synthetic)

围绕原生事件的 跨浏览器的事件对象，不同浏览器的表现相同行为。 nativeEvent为 内置原生事件对象。

如果绑定事件函数是一个普通定义原型的函数，那么this是undefined

那么为了可以使用事件绑定。 可以使用bind(this)或箭头函数来处理。

通过event.stopPropagation 阻止事件传播(冒泡和捕获)。event.stopImmediatePropagation阻止事件传播(包括自身的绑定其他函数传播)

capture

机制

react17及以后，对容器的id=root对捕获和冒泡都进行了委托。 react17之前则只是对document的冒泡阶段进行了委托。

合成机制并没有给元素本身进行事件绑定，而是给元素设置了attrs，等待#root触发事件来进行委托处理。

evenet.persist

react16合成事件内部基于”事件对象池”缓存机制。 触发事件，赋值对象成员，操作完成后，清空成员信息。 可能导致 定时器等操作后，事件对象成员信息丢失问题。

可以在 react16将 合成事件对象成员信息 保留下来。 解决问题。

事件细节知识

  1. 移动端有300ms延迟，且双击不会触发点击事件。
  2. 解决300ms可以使用手写touch事件(利用偏移量)
  3. 利用faskClick插件解决移动端300ms延迟点击问题。

Q: 循环给元素增加事件好不好?

A: 没区别，绑定合成事件本身就是基于事件委托，不需要自己委托。 而vue则需要手动委托一下。

hooks索引

函数组件的每一次更新/渲染都是把函数重新执行了一次。 产生新的”私有上下文”，内部代码重新执行。

不要在循环、条件、嵌套函数中调用hooks。

只在React函数组件中使用hooks或者是自定义hooks中调用。

useState

函数组件使用状态。更新视图。

值是一个数组，第一项为值，第二项为修改状态值的方法。 调用第二项的方法传参来修改state。

setVal改变对象

setVal 是设置整体val的值。 如果使用对象则是需要扩展展开。

多次使用useState

如果有多个state需要使用，则需要给每一个state来进行use。

同异步

Q: 多次同时执行useState修改，组件会执行几次？
A: 一次，等同于Class组件的 setState。

Q: 怎么同步执行SetState？
A: react-dom的 flushSync 来进行更新updater队列。

Q: react16.x的 同异步 等同于 setState 一样嘛？
A: 是的，放在合成事件为异步，放在定时器、事件监听器内是同步的。

循环调用及优化机制

如果想要循环 自身+1 ，但是上下文的state永远都是第一次的初始值。

如下永远都是批更新一次，且由于上下文的approveNum永远都是初始值10, 所以执行完永远都是视图更新一次， 且为11

useState自带了性能优化，类似于 pureComponent的 Object.is比较，一致的话是不会进行更新。

如何拿到循环上次 的 变化的state， 可以同 setState的 callback => prevState来进行操作(同class 的 setState(prevState => state))

代码优化： 当使用数据初次计算完成后，以后的试图更新等操作后 不需要再执行第二次。那么就需要优化此部分代码。(惰性处理)

useReducer

当前组件内的状态管理。

创建reducer(state, action)=>state

解构userReducer(reducer, initState) = [state, dispatch]

然后使用state， dipatch派发state。

useEffect

函数组件的生命周期。

只能在函数作用域内调用，不可以被条件判断、循环等代码块包裹。

useEffect(callback)

第一次渲染完成执行callback， 等价于 componentDidMount。

组件每一次更新完毕，也会callback， 等价于 componentDidUpdate。

useEffect(callback, [])

第一次渲染完成执行callback。 每一次更新则不会执行。

useEffect(callback, […states])

数组内放入依赖，数组内state发生更新会触发callback。

useEffect(callback:function)

callback返回一个新的函数。

在组件销毁时候执行，拿到状态都是上一次的状态。更新时候，销毁上一次组件，触发return的回调。

异步请求

callback必须返回一个函数，所以不能使用async来修饰。

解决办法： 可以使用.then。 也可以使用自执行函数。

useLayoutEffect

会在创建virtualDOM创建后，通知执行。

阻止渲染真实dom, 执行effect的callback。

和useEffect区别：

  • useLayoutEffect优先执行
  • 都是可以获取DOM，区别是浏览器是否渲染
  • useEffect会重新绘制，useLayoutEffect会执行回调后，决定是否重新绘制。

useRef

函数组件使用ref也是可以(但是不推荐)使用变量赋值来实现。

也可以基于react.createRef()来实现。

使用useRefhooks函数实现。(只可以在函数组件使用。)

useRef与createRef

useRef 每次组件更新都是使用的第一次创建好的ref对象。

createRef每次函数组件更新都会创建一个新的ref对象。

useImperativeHandle

ref想要获取函数组件的 状态/方法，而不是一个简单的dom元素

返回一个对象，父组件可以获取到对象。 返回数组则是数组。 父组件通过 [ref].current来访问数据

useMemo计算属性-优化

类同于useEffect的计算缓存属性：

let variable = useMemo((any):any, [])

useCallback-优化

函数组件内每次更新都会重新创建 堆内存。

let [function] = useCallback( ()=>void, [variable] )

如果没有设置依赖时候，则一直是第一次创建出来的堆内存。

如果设置了依赖，依赖变化则引用地址也会变化。

useCallback也不可以随意使用。具体场景如下可以酌情使用:

父组件属性/函数传递给子组件。(每次传入不同引用地址，则每次都会重新更新。

  1. 使用useCallback
  2. 子组件(class)可以继承React.pureComponent， 进行浅比较
  3. 函数组件可以使用React.memo(componentChild) 进行浅比较
