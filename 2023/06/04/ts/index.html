什么是ts?

  2. 编译为js代码

安装

自动编译

类型声明(类型注解)是一种轻量的为函数或变量添加约束的方式。

基础

原始数据类型

  1. boolean
  2. number
  3. string
  4. undefined
  5. null

通常undefined和null可以作为其他类型子类型(严格模式不允许)

数组

  1. 类型数组
  2. 数组泛型

对象

表示非原始数据类型：除number、string、boolean之外的类型。

大写的Object和字面量{}(尽量别用)可以表示所有类型。

以上都是可以的(非严格)。

any & void

  1. any是任何类型。
  2. void是空值(表示没有任何返回值或类型)

unknown

  1. 只能赋值给自身或者any类型。
  2. 不可以访问属性和方法

never

那些根本不会有返回值的函数、抛出的错误。

类型推断

没有明确指定类型，ts会推测一个类型。

  1. 定义赋值时，定义为相应的类型
  2. 定义不赋值，默认定义为any类型

联合类型

可以取多种特定类型。

|关键符

访问属性、方法必须是联合类型共有属性和方法。 再次赋值会类型推断来定义其中类型。

交叉类型

& 关键字创建。

内置对象类型

  1. Number
  2. Date
  3. RegExp
  4. Error
  5. XMLHttpRequest

DOM

  1. HTMLElement
  2. Element
  3. HTML + 标签名称 + Element

BOM(浏览器内类型)

  1. Storage
  2. Location
  3. Promise

接口

是对行为的抽象，ts用于对象形状进行描述。

约束形状与接口一致

注意:

  1. 接口首字母大写
  2. 某些编程语言接口推荐加I前缀
  3. 约束目标缺少属性是不允许的
  4. ?表示可选属性
  5. 任意属性通过[propName:string]:T来定义
  6. 任意属性定义，已知属性必须是任意属性类型的子类型(推荐使用联合类型)
  7. 一个接口只能定义一个任意属性

接口描述数组

不常用

接口描述函数

函数类型

基本写法：

可选/默认/剩余参数

可选参数用?关键符。
默认参数使用arg1: string = 'liuxiao'

注意

  1. 可选参数后不允许再添加必选参数
  2. 默认参数可以放在可选、必选参数后面

剩余参数/重载

函数重载: 函数名相同，形参、返回值不同需要定义不同的函数类型(函数类型定义列表)。

注意

  1. 实现签名必须是兼容重载签名的

类型断言

  1. 通过as关键字 var as number
  2. 通过<>关键符 <number>var

jsx(react)中必须使用前者，即as。

联合类型断言

任何类型断言为any

as any极有可能掩盖真正的错误！轻易不要使用。
但是严格性和开发便利性之间可以做到一个平衡。

any断言为具体类型

可能是第三方库、别人遗留代码、垃圾代码等。我们可以将他断言为具体的类型

进阶

类型别名

通过type关键字来创建类型别名。

高级用法：使用extends关键字。 是包含的意思。

一般用于联合类型

结合泛型

字符串字面量类型

使用type关键字 用来约束只能是 某几个字符串中 其中一个。

元组(Tuple)

Tuple元定义 合并不同类型对象。

可以理解为 定义数组每一项的type且越界元素必须为 定义过的联合类型

枚举(enum)

枚举成员默认赋值从0开始递增数字，同时对枚举名进行反向映射。

注意:

  1. 手动赋值与默认赋值重复，ts不会察觉，只会覆盖
  2. 手动赋值也可以不是数字，可以通过类型断言覆盖(必须是最后一项)
  3. 手动赋值可以为小数，而后递增仍为1
  4. 手动赋值不为number类型，不会反向映射。

常数项、计算所得项

常数项: 默认值和手动赋值
计算所得项: { two='string'.length }

注意

  1. 计算所得项后无法定义默认值！
  2. 计算所得项可以手动赋值

常数枚举

const enum x{} 关键字创建。

注意

  1. 编译后被删除无法查看，可以打印出枚举值
  2. 不能包含计算所得项(可以手动赋值)

外部枚举

declare enum x{} 关键字创建。

注意

  1. 编译后被删除无法被查看
  2. 常常出现在声明文件

类

定义类时候，需要定义下属性。

继承

  1. extends继承
  2. 不需要写属性类型，super()传入即可
  3. 可以重写父类方法
  4. super.method()调用父类方法

存取器

getter和setter对属性读取和赋值行为改写。

静态成员

类自己的属性和方法，不会被实例化。

static关键字创建

public、private、protected修饰符

  • public: 默认的, 成员为公有的，可以在任何地方被访问。
  • private: 私有,不可以在类之外访问。(子类只继承，不可访问)
  • protected:私有,不可以在类之外访问。(子类可以继承、可以访问)

readonly

只读属性。 constructor内可以修改。

参数属性

创建并初始化参数属性

抽象类

  1. abstract关键字创建。
  2. 不能被实例化。
  3. 抽象方法必须被子类实现。
  4. 抽象方法只能定义在抽象类中。

类的类型

extends 关键字继承。 继承多个接口可以用,分割

extends关键字继承. 接口继承类的 实例属性和方法.
