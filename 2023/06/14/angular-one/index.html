angular?

Angular是基于TS的开发平台。 核心知识: 组件、模板、依赖注入。

组件包含import {Component} from '@angular/core' @Component{ selector: '#id', template: ''}装饰器的TS类

模板内的{{}}、<p [id]='SayHi'></p>、<p (click)="SayHi()"></p>、*ngIf、*ngFor

依赖注入的@Injectable({})、constructor(private logger:Logger)

自带一些库： 路由器、表单、Angular HttpClient(基于XMLHttpRequest)、动画、PWA(processive web application)、原理图(搭建脚手架、重构、升级的自动化工具)

循环渲染

使用*ngFor(结构体指令)来循环数组渲染。通过{{}}模板插值，渲染为文本。

*ngFor类似于let value of arr的语法。

动态绑定(attrs)

使用[]=""动态绑定属性

条件渲染

使用*ngIf条件判断渲染等。

事件绑定

使用(eventName)="Function()"给元素绑定事件

生成组件

使用一下注册一个子组件。 生成器生成三个文件为:

  • product-alerts.component.ts

  • product-alerts.component.html

  • product-alerts.component.css

  • product-alerts.component.spec.ts

在.ts文件中，引入Component。

@Component()装饰器会

  1. 指出它后面类为组件，包含了选择器、模板、样式
  2. 导出了类ProductAlertsComponent，处理组件功能。
父组件 => 子组件

子组件通过引入Input, 使用@Input()装饰器定义数据属性， 这个装饰器指出 此属性值是从父组件传递而来。 然后子组件就可以使用数据。

而父组件需要通过[]=""传入动态属性数据。

AppModule.ts会在declarations:[]声明组件，来确保可以在应用内使用子组件。

子组件 => 父组件

子组件向父组件通信。

  • 在子组件的@angular/core内引入Output、EventEmitter符号，并使用@Output定义子组件一个notify = new EventEmitter()事件。

  • 父组件ts内，设置一个onNotify自己的事件, 并定义操作。

  • 父组件html内，给子组件定义绑定自定义事件, 事件名与子组件定义的eventEmitter实例化对象一致，事件函数是父组件定义函数。(notify)="onNotify()"
