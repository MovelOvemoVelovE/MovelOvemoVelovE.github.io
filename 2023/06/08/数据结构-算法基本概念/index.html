数据结构-算法基本概念

发表于2023-06-08|更新于2023-06-08|数据结构
|阅读量:

什么是算法?

对特定问题求解的一种描述， 是指令的有限序列，每条指令可以表示一个、多个操作。

特性

1. 有穷性: 一个算法必须在执行有穷步之后结束，且每一个都在有穷时间内完成。

2. 确定性: 必须有确切含义， 相同输入必须有着相同输出。

3. 可行性: 算法描述的操作可以通过 已经实现的 基本运算执行有限次来实现

4. 输入和输出: 有输入(取自某个特定数据对象集合)和输出(与输入有特定关系的量)

好算法的特质

  1. 正确性，能够正确求解
  2. 可读性，良好可读性，可以帮助理解
  3. 健壮性，输入非法数据，适当反应、处理，不会莫名其妙的输出结果。
  4. 高效率和低存储量需求， 时间复杂度低和空间复杂度低。

算法效率

算法效率就是 时间复杂度 和 空间复杂度。

评估时间开销

Q: 如何评估时间开销?

A: 是让算法先运行，事后统计时间?

会存在问题:

  1. 与机器性能有关
  2. 和编程语言有关(越高级的语言效率越低)
  3. 编程机器指令质量有关
  4. 有些算法做不到事后统计(导弹等)

所以是不太可行的。 于是有 算法时间复杂度

时间复杂度

时间复杂度: 事前预估算法时间(T)与问题规模n之间的关系。 T = T(n)

这个例子的T与n关系为: T(n) = 3n + 3 。

那么如果是我们必须要一行一行的数代码执行次数吗?，需要是否可以忽略表达式某些部分/简化表达式呢

上面例子当规模足够大时，第一次简化只考虑阶数高的部分，忽略掉低阶函数。

第二次用大O表示同阶/同等数量级也就是只考虑他的数量级就可以了当 n -> ∞, 二者之比为常数.也就是:

$$
\lim_{n\rightarrow\infty}\frac{T(n)}{O(n)} = k
$$

保留规则是:

  1. 加法规则： 只保留最大阶数
  2. 乘法规则: 都保留。

最大阶数规则表:
O(1) < O(log₂n) < O(n) < O(nlog₂n) < O(n²) < O(n³) < O(2ⁿ) < O(n!) < O(nⁿ)
“常对幂指阶”

最好、最坏、平均

搜索一个数组内是否存在某项，有则跳出循环，那么就会出现最好、最坏、平均时间复杂度(第一项、最后一项、中间项出现的情况)。

一般只判断最坏、平均复杂度标准。

空间复杂度

无论问题规模，内存空间都为常量。 空间复杂度为: S(n)=O(n): s => space.
